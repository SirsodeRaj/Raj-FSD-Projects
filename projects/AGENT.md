## 🤖 Agent Instructions

### When Starting a New Session

1. **Read this file first** — Always read `AGENT.md` at the project root before doing any work.

2. **Check current progress** — List the `projects/` directory to verify completed projects against the tracking table above.

3. **Identify the next project** — Build the next uncompleted project from the list in sequential order (start from #45 unless directed otherwise).

4. **Create project folder** — Use kebab-case naming convention (e.g., `meme-viewer`, `css-gradient-generator`).

5. **Build the project** — Create a complete, functional web application with:
   - `index.html` — Well-structured HTML with semantic elements
   - `style.css` — Clean, responsive CSS with modern design
   - `script.js` — Functional JavaScript with proper error handling
   - `README.md` — Brief description, features, and how to run

6. **Update this file** — After completing a project:
   - Add the new project to the **Completed Projects** table
   - Increment the progress counter
   - Remove or mark off the project from the **Next Projects to Build** list

7. **Follow best practices**:
   - Use semantic HTML5 elements
   - Mobile-responsive design
   - Clean, modern UI with smooth animations
   - Well-commented code
   - No external dependencies unless specified
   - Store data in `localStorage` where applicable

### Project Structure Template

```
projects/
└── project-name/
    ├── index.html
    ├── style.css
    ├── script.js
    └── README.md
```

### Code Quality Standards

- **HTML:** Semantic tags, proper accessibility attributes, meta viewport
- **CSS:** CSS variables for theming, flexbox/grid layouts, media queries, smooth transitions
- **JS:** ES6+ syntax, modular code, event delegation, localStorage for persistence
- **Design:** Modern, visually appealing, consistent color palette, good typography

### Progress Tracking Format

When updating the completed projects table, use this format:

```
| # | folder-name | Project Idea # | Project Name | Level | Status |
```
